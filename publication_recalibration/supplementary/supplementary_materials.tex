\subsection{S1.1 Zero-Shot Classification}

A pre-trained language model classified severity without task-specific training.

\[
\hat{y} = \arg\max_{c} \; P(\text{entailment} \mid \text{description}, c)
\]
where $P(\text{entailment} \mid \text{description}, c)$ = probability that the Natural Language Inference (NLI) model outputs ``entailment'' given the Drug-Drug Interaction (DDI) description as premise and severity label $c$ as hypothesis; $c \in \{$Contraindicated, Major, Moderate, Minor$\}$.

\medskip
\noindent\textbf{Algorithm 1:} Zero-Shot Classification
\begin{verbatim}
FUNCTION ZeroShotClassify(description)
  INPUT: DDI description text
  OUTPUT: Severity label
  
  FOR EACH class IN {Contraindicated, Major, Moderate, Minor}
    score[class] <- ComputeEntailment(description, class)
  END FOR
  predicted <- ArgMax(score)
  confidence <- Max(score)
  IF predicted IN {Contraindicated, Major} AND confidence < 0.65 THEN
    predicted <- DowngradeOneLevel(predicted)
  END IF
  RETURN predicted
END FUNCTION
\end{verbatim}

\subsection{S1.2 Rule-Based Classifier}

Severity was assigned based on weighted keywords learned from training data.

\[
w_k = \ln\left(\frac{P(k|\text{Major})}{P(k|\text{Moderate})}\right)
\]
where $P(k|\text{Major})$ = frequency of keyword $k$ in Major-severity DDIs / total Major DDIs; $P(k|\text{Moderate})$ = frequency in Moderate-severity DDIs / total Moderate DDIs.

Score ($\mathbb{1}[\cdot]$ = indicator function, 1 if keyword present, 0 otherwise):
\[
S = \sum_{k} w_k \cdot \mathbb{1}[k \in \text{description}]
\]

\medskip
\noindent\textbf{Algorithm 2:} Rule-Based Classification
\begin{verbatim}
FUNCTION TrainWeights(labeled_data)
  INPUT: Training set with severity labels
  OUTPUT: Keyword weight dictionary W
  
  FOR EACH keyword k IN vocabulary
    W[k] <- Log(FreqMajor(k) / FreqModerate(k))
  END FOR
  RETURN W
END FUNCTION

FUNCTION RuleBasedClassify(description, W)
  INPUT: DDI description, weight dictionary W
  OUTPUT: Severity label
  
  score <- 0
  FOR EACH keyword k IN description
    IF k IN W THEN
      score <- score + W[k]
    END IF
  END FOR
  IF score >= Percentile96 THEN RETURN Contraindicated
  ELSE IF score >= Percentile92 THEN RETURN Major
  ELSE IF score <= Percentile2 THEN RETURN Minor
  ELSE RETURN Moderate
  END IF
END FUNCTION
\end{verbatim}

\medskip
\noindent Thresholds (P96, P92, P2 = percentiles from training score distribution):
\[
\text{Severity} = 
\begin{cases}
\text{Contraindicated} & S \geq 96\text{th percentile} \\
\text{Major} & S \geq 92\text{nd percentile} \\
\text{Minor} & S \leq 2\text{nd percentile} \\
\text{Moderate} & \text{otherwise}
\end{cases}
\]

\subsection{S1.3 Evidence-Based Variant}

Identical to rule-based, with a penalty for low-confidence predictions.

$S_{\text{adj}} = 0.8 \cdot S_{\text{raw}}$ when confidence $< 0.5$.

\medskip
\noindent\textbf{Algorithm 3:} Evidence-Based Classification
\begin{verbatim}
FUNCTION EvidenceBasedClassify(description, W)
  INPUT: DDI description, weight dictionary W
  OUTPUT: Severity label
  
  score <- RuleBasedScore(description, W)
  confidence <- MatchedKeywords / TotalKeywords
  IF confidence < 0.5 THEN
    score <- 0.8 * score
  END IF
  RETURN ClassifyByThresholds(score)
END FUNCTION
\end{verbatim}

\section{S2: Statistical Methods}
\label{sec:s2}

\subsection{S2.1 Evaluation Metrics}

Standard classification metrics measured model performance.

Accuracy (proportion of correct predictions):
\[
\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}
\]

Precision (positive predictive value):
\[
\text{Precision} = \frac{TP}{TP + FP}
\]

Recall (sensitivity):
\[
\text{Recall} = \frac{TP}{TP + FN}
\]

F1-score (harmonic mean of precision and recall):
\[
F_1 = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}
\]

where $TP$ = true positives, $TN$ = true negatives, $FP$ = false positives, $FN$ = false negatives.

\bigskip
\subsection{S2.2 McNemar's Test}

McNemar's test compared paired classifiers on the same dataset to determine whether their disagreements were statistically significant.

\begin{table}[h]
\centering
\caption{Contingency table for McNemar's test.}
\begin{tabular}{@{}lcc@{}}
\toprule
 & Zero-Shot Correct & Zero-Shot Wrong \\
\midrule
Rule-Based Correct & 502 & 6,900 \\
Rule-Based Wrong & 1,962 & 1,786 \\
\bottomrule
\end{tabular}
\end{table}

\[
\chi^2 = \frac{(|b - c| - 1)^2}{b + c} = \frac{(6900 - 1962 - 1)^2}{8862} = 2750.4 \quad (p < 10^{-16})
\]
where $b$ = cases where only Rule-Based was correct, $c$ = cases where only Zero-Shot was correct.

\bigskip
\subsection{S2.3 Confidence Intervals}

Wilson score intervals estimated the range of plausible accuracy values given sample size.

Wilson score 95\% Confidence Interval (CI):
\[
\text{CI} = \frac{\hat{p} + \frac{z^2}{2n} \pm z\sqrt{\frac{\hat{p}(1-\hat{p})}{n} + \frac{z^2}{4n^2}}}{1 + z^2/n}
\]
where $\hat{p}$ = observed proportion, $n$ = sample size, $z$ = 1.96 for 95\% CI.

\bigskip
\subsection{S2.4 Agreement Statistics}

Cohen's kappa measured inter-rater agreement corrected for chance.

Cohen's $\kappa$:
\[
\kappa = \frac{p_o - p_e}{1 - p_e}
\]
where $p_o$ = observed agreement proportion, $p_e$ = expected agreement by chance.

Statistical significance:
\[
z = \kappa \,/\, \text{SE}_{\kappa}
\]
where $\text{SE}_{\kappa}$ = standard error of kappa.

\bigskip
\subsection{S2.5 Additional Metrics}

Jensen-Shannon divergence quantified similarity between predicted and reference severity distributions.

Jensen-Shannon divergence:
\[
D_{JS} = \tfrac{1}{2} D_{KL}(P \| M) + \tfrac{1}{2} D_{KL}(Q \| M), \quad M = (P + Q)/2
\]
where $P$ = predicted distribution, $Q$ = reference distribution, $M$ = mixture distribution, and $D_{KL}(A \| B) = \sum_i A_i \log(A_i / B_i)$ is the Kullback-Leibler divergence.

Chi-square goodness of fit tested whether observed frequencies differed significantly from expected frequencies:
\[
\chi^2 = \sum_i \frac{(O_i - E_i)^2}{E_i}
\]
where $O_i$ = observed frequency for category $i$, $E_i$ = expected frequency for category $i$.

\bigskip
\subsection{S2.7 DDInter Validation Metrics}

To validate our severity classifications against DDInter ground truth labels, we employed three complementary metrics:

\textbf{Exact Accuracy:} The percentage of predictions matching DDInter's severity label exactly:
\[
\text{Exact Accuracy} = \frac{1}{N} \sum_{i=1}^{N} \mathbf{1}[\hat{y}_i = y_i]
\]
where $\hat{y}_i$ is the predicted severity and $y_i$ is DDInter's ground truth label for DDI pair $i$.

\textbf{Adjacent Accuracy:} The percentage of predictions within $\pm 1$ severity level of ground truth:
\[
\text{Adjacent Accuracy} = \frac{1}{N} \sum_{i=1}^{N} \mathbf{1}[|\hat{y}_i - y_i| \leq 1]
\]
where severity levels are encoded as integers (Minor = 0, Moderate = 1, Major = 2). This metric accounts for the ordinal nature of severity classifications and captures clinically acceptable near-matches.

\textbf{Cohen's $\kappa$:} Inter-rater agreement beyond chance:
\[
\kappa = \frac{p_o - p_e}{1 - p_e}
\]
where $p_o$ is the observed agreement (exact accuracy) and:
\[
p_e = \sum_{c} P(\hat{y} = c) \cdot P(y = c)
\]
is the expected agreement by chance, computed as the sum over all classes $c$ of the product of marginal probabilities.

\noindent Interpretation of $\kappa$ values:
\begin{itemize}
    \item $\kappa > 0$: Agreement better than random chance
    \item $\kappa = 0$: Chance-level agreement
    \item $\kappa < 0$: Systematic disagreement (worse than chance)
\end{itemize}

\section{S3: Knowledge Graph Construction}
\label{sec:s3}

The DDI knowledge graph integrates data from multiple biomedical databases using fact-based entity linking with exact identifier matching.

\begin{figure}[H]
\centering
\includegraphics[width=0.85\textwidth]{er_diagram.png}
\caption{\textbf{Entity-Relationship diagram of the DDI knowledge graph schema.} The diagram shows the primary entities (Drug, Protein, Side Effect, Disease, Pathway) and their relationships. Drug nodes are linked to other drugs via DDI edges with severity attributes. Drugs connect to proteins through target, enzyme, carrier, and transporter relationships. Side effects and diseases are linked via SIDER and CTD databases respectively.}
\label{fig:er_diagram}

\subsection{S3.1 Data Sources}

\begin{itemize}
    \item \textbf{DrugBank XML}: Drug properties, targets, enzymes, carriers, transporters, pathways, categories, DDIs
    \item \textbf{SIDER}: Side effect profiles (matched by drug name)
    \item \textbf{CTD}: Drug-disease associations (matched by CAS number or drug name)
\end{itemize}

\subsection{S3.2 Entity Matching Strategy}

Entities are linked using exact identifier matching (no similarity-based matching):
\begin{enumerate}
    \item DrugBank ID (primary identifier)
    \item Drug Name (exact case-insensitive match)
    \item CAS Number (chemical identifier)
    \item ATC Code (therapeutic classification)
\end{enumerate}

\medskip
\noindent\textbf{Algorithm 10:} Knowledge Graph Construction
\begin{verbatim}
FUNCTION BuildKnowledgeGraph(drugbank_xml, sider_data, ctd_data)
  INPUT: DrugBank XML, SIDER side effects, CTD associations
  OUTPUT: Knowledge graph G = (V, E)
  
  // Phase 1: Parse DrugBank XML
  drugs <- ParseDrugBankXML(drugbank_xml)
  FOR EACH drug IN drugs
    ExtractIdentifiers(drug)  // DrugBank ID, name, CAS, ATC
    ExtractTargets(drug)      // Protein targets
    ExtractEnzymes(drug)      // Metabolizing enzymes
    ExtractTransporters(drug) // Drug transporters
    ExtractPathways(drug)     // Biological pathways
    ExtractDDIs(drug)         // Drug-drug interactions
  END FOR
  
  // Phase 2: Link external data sources
  FOR EACH drug IN drugs
    // SIDER: Side effect matching
    IF ExactNameMatch(drug.name, sider_data) THEN
      drug.side_effects <- LinkSiderData(drug.name)
    END IF
    
    // CTD: Disease association matching
    IF drug.cas_number IN ctd_data OR ExactNameMatch(drug.name, ctd_data) THEN
      drug.disease_associations <- LinkCTDData(drug)
    END IF
  END FOR
  
  // Phase 3: Build graph structure
  G <- InitializeMultiDiGraph()
  
  // Add nodes by type
  FOR EACH drug IN drugs
    AddNode(G, drug.id, type="drug", attributes=drug.properties)
  END FOR
  FOR EACH protein IN proteins
    AddNode(G, protein.id, type="protein", attributes=protein.properties)
  END FOR
  FOR EACH side_effect IN side_effects
    AddNode(G, "SE:" + side_effect.id, type="side_effect")
  END FOR
  FOR EACH disease IN diseases
    AddNode(G, "DIS:" + disease.id, type="disease")
  END FOR
  FOR EACH pathway IN pathways
    AddNode(G, pathway.id, type="pathway")
  END FOR
  
  // Add edges with provenance
  FOR EACH ddi IN drug_interactions
    AddEdge(G, ddi.drug1, ddi.drug2, 
            type="interacts_with",
            severity=ddi.severity,
            description=ddi.description,
            provenance="DrugBank")
  END FOR
  FOR EACH (drug, target) IN drug_targets
    AddEdge(G, drug.id, target.id,
            type="targets",
            action=target.action,
            provenance="DrugBank")
  END FOR
  // ... additional edge types for enzymes, transporters, 
  //     side effects, diseases, pathways
  
  RETURN G
END FUNCTION
\end{verbatim}

\medskip
\noindent\textbf{Algorithm 11:} Severity Recalibration Integration
\begin{verbatim}
FUNCTION IntegrateSeverityRecalibration(G, severity_model)
  INPUT: Knowledge graph G, trained severity classifier
  OUTPUT: Graph G with recalibrated severity labels
  
  FOR EACH edge IN G.edges WHERE edge.type = "interacts_with"
    description <- edge.description
    
    // Apply rule-based severity classifier
    score <- ComputeKeywordScore(description, severity_model.weights)
    severity <- ClassifyByThresholds(score)
    
    // Update edge attributes
    edge.original_severity <- edge.severity
    edge.severity <- severity
    edge.severity_score <- score
    edge.recalibrated <- TRUE
  END FOR
  
  RETURN G
END FUNCTION
\end{verbatim}

\subsection{S4.1 Betweenness Centrality}

Betweenness centrality quantifies how often a drug lies on the shortest path between other drug pairs in the network. Drugs with high betweenness centrality act as ``bridges'' within the interaction network and reflect indirect effects between different drug clusters. Local anesthetics such as Procaine and Lidocaine, as well as the biologics Sotatercept and Abciximab, showed high betweenness. These agents interacted with multiple drug classes and bridged distinct therapeutic groups, respectively.

\subsection{S4.2 Eigenvector Centrality}

Eigenvector centrality emphasizes not only the number of connections a drug has but also the importance of its neighboring nodes. A drug with a high score tends to interact with other highly connected drugs. Antiarrhythmics such as Quinidine and cardiovascular agents including Propranolol, Verapamil, and Isradipine were at the top eigenvector centrality rankings, which showed their tendency to interact with other high-risk medications.

\subsection{S4.3 PageRank Centrality}

PageRank centrality measures how likely a drug is to be reached through random traversal through the interaction network. It serves as an indicator of high-interaction targets, which are most likely to be affected when new drugs are added to a regimen. A drug has high PageRank if many other drugs interact with it, especially if those drugs themselves have many interactions. Local anesthetics (Procaine, Lidocaine, Benzocaine) exhibited the highest PageRank scores, which indicated their extensive interaction profiles with multiple drug classes.

\subsection{S4.4 Louvain Community Detection}

The Louvain algorithm is a greedy optimization method for detecting community structure in large networks. It operates in two iterative phases:

\textbf{Phase 1 (Local Moving):} Each node is initially assigned to its own community. For each node $i$, the algorithm calculates the modularity gain from moving $i$ to each neighboring community $C$:
\[
\Delta Q = \left[ \frac{\Sigma_{in} + 2k_{i,in}}{2m} - \left( \frac{\Sigma_{tot} + k_i}{2m} \right)^2 \right] - \left[ \frac{\Sigma_{in}}{2m} - \left( \frac{\Sigma_{tot}}{2m} \right)^2 - \left( \frac{k_i}{2m} \right)^2 \right]
\]
where $\Sigma_{in}$ is the sum of weights inside community $C$, $\Sigma_{tot}$ is the total degree of nodes in $C$, $k_i$ is the degree of node $i$, $k_{i,in}$ is the sum of weights from $i$ to nodes in $C$, and $m$ is the total edge weight of the graph.

\textbf{Phase 2 (Aggregation):} Communities are collapsed into super-nodes, creating a new network where edge weights between super-nodes equal the sum of inter-community edges.

These phases repeat until no further modularity improvement is possible, optimizing the global modularity:
\[
Q = \frac{1}{2m} \sum_{ij} \left[ A_{ij} - \frac{k_i k_j}{2m} \right] \delta(c_i, c_j)
\]

\section{S6: Polypharmacy Risk Index (PRI) Derivation}
\label{sec:s6}

The Polypharmacy Risk Index (PRI) quantifies overall interaction risk for individual drugs based on network topology and severity profiles:

\[
\text{PRI}(d) = 0.25 \cdot C_{\text{degree}}(d) + 0.30 \cdot C_{\text{weighted}}(d) + 0.20 \cdot C_{\text{betweenness}}(d) + 0.25 \cdot S(d)
\]

\textbf{Degree centrality} $C_{\text{degree}}(d)$ measures how many potential interactions a drug possesses. It captures the interaction burden as the number of drugs interacting with the drug, normalized by the maximum possible degree in the network. The 0.25 weight was specified based upon the fundamental importance of interaction count as a baseline risk indicator, while avoiding over-emphasis without considering severity.

\textbf{Weighted degree centrality} $C_{\text{weighted}}(d)$ extends the simple degree measure by incorporating severity information. It was calculated as the severity-weighted sum of all interactions and was assigned a weight of 0.30 in the Polypharmacy Risk Index.

\textbf{Betweenness centrality} $C_{\text{betweenness}}(d)$ quantifies the role of a drug in risk propagation. Drugs with high betweenness centrality serve as ``bridges'' between interaction clusters and may facilitate cascade effects when multiple medications are combined. The low weight assignment (0.20) reflects the fact that direct interaction measures are more immediately relevant to individual patient risk assessment, compared to system-level insights into potential cascade effects derived from network topology.

\textbf{Severity profile score} $S(d)$ measures the proportion of interactions of a drug that were classified as contraindicated or of major severity (weight: 0.25). This metric identifies drugs that, while potentially having fewer total interactions, are predominantly involved in high-risk combinations. The weight of 0.25 was assigned to ensure that drugs with fewer but predominantly dangerous interactions were appropriately flagged.

Contraindicated interactions received the highest severity weight of 10.0 in the weighted degree calculation, followed by major interactions (7.0), moderate interactions (4.0), and minor interactions (1.0) (see S7 for details).

\subsection{S7.2 Alternative Recommendation Score (ARS)}

Candidates meeting the filtering criteria are ranked by the Alternative Recommendation Score (ARS), which combines normalized regimen-specific severity reduction with network-derived Polypharmacy Risk Index improvement:

\[
\text{ARS}(d_{\text{alt}}) = 0.70 \times \text{SeverityReduction}(d_{\text{alt}}) + 0.30 \times \text{PRI\_Improvement}(d_{\text{alt}})
\]

\textbf{Severity Reduction} (weight: 0.70) quantifies the decrease in total severity-weighted interactions when the alternative drug replaces the original within the current regimen. This component receives the dominant weight because it directly measures immediate DDI risk reduction relevant to patient safety. Higher values indicate that the alternative drug has fewer or less severe interactions with other medications in the patient's regimen.

\textbf{PRI Improvement} (weight: 0.30) captures the network-level risk reduction when substituting the drug, considering the alternative's overall interaction profile across the entire knowledge graph. This component supports long-term optimization by identifying alternatives that have generally safer interaction profiles beyond the immediate regimen context.

Higher ARS scores indicate safer substitutions; candidates are ranked in descending order to prioritize alternatives offering the greatest combined benefit in both regimen-specific and network-wide risk reduction.

\subsection{S7.3 Severity Reduction Component (Weight: 70\%)}

Quantifies the decrease in total severity-weighted interactions when the alternative drug replaces the original within the current regimen.

\section{S8: Algorithm Pseudocode}
\label{sec:s8}

\subsection{S8.1 ATC-Based Candidate Filtering}

\begin{verbatim}
FUNCTION FindCandidates(drug_to_replace, knowledge_graph)
  INPUT: drug_to_replace (DrugBank ID), knowledge_graph
  OUTPUT: List of candidate alternatives
  
  original_atc <- GetATCCodes(drug_to_replace)
  original_atc_level3 <- ExtractLevel3(original_atc)
  candidates <- []
  
  FOR EACH drug IN knowledge_graph.drugs
    IF drug.id != drug_to_replace THEN
      drug_atc <- GetATCCodes(drug.id)
      drug_atc_level3 <- ExtractLevel3(drug_atc)
      IF HasOverlap(original_atc_level3, drug_atc_level3) THEN
        candidates.append(drug)
      END IF
    END IF
  END FOR
  RETURN candidates
END FUNCTION
\end{verbatim}

\subsection{S8.2 Severity Reduction Calculation}

\begin{verbatim}
FUNCTION CalculateSeverityReduction(original, alternative, regimen, kg)
  INPUT: original_drug, alternative_drug, regimen, knowledge_graph
  OUTPUT: Normalized severity reduction score [0, 1]
  
  weights <- {Contraindicated: 10, Major: 7, Moderate: 4, Minor: 1}
  
  original_severity <- 0
  FOR EACH other_drug IN regimen
    interaction <- kg.GetInteraction(original, other_drug)
    IF interaction EXISTS THEN
      original_severity += weights[interaction.severity]
    END IF
  END FOR
  
  alternative_severity <- 0
  FOR EACH other_drug IN regimen
    interaction <- kg.GetInteraction(alternative, other_drug)
    IF interaction EXISTS THEN
      alternative_severity += weights[interaction.severity]
    END IF
  END FOR
  
  IF original_severity > 0 THEN
    RETURN Max(0, (original_severity - alternative_severity) / original_severity)
  ELSE
    RETURN 0
  END IF
END FUNCTION
\end{verbatim}

\subsection{S8.3 Alternative Recommendation Score}

\begin{verbatim}
FUNCTION CalculateARS(original, alternative, regimen, kg)
  INPUT: original_drug, alternative_drug, regimen, knowledge_graph
  OUTPUT: ARS score [0, 1]
  
  w_severity <- 0.70
  w_pri <- 0.30
  
  severity_reduction <- CalculateSeverityReduction(original, alternative, 
                                                    regimen, kg)
  
  pri_original <- kg.GetPRI(original)
  pri_alternative <- kg.GetPRI(alternative)
  IF pri_original > 0 THEN
    pri_improvement <- Max(0, (pri_original - pri_alternative) / pri_original)
  ELSE
    pri_improvement <- 0
  END IF
  
  RETURN w_severity * severity_reduction + w_pri * pri_improvement
END FUNCTION
\end{verbatim}

\subsection{S8.4 Find Best Alternatives}

\begin{verbatim}
FUNCTION FindBestAlternatives(drug_to_replace, regimen, kg, top_k=5)
  INPUT: drug_to_replace, regimen, knowledge_graph, top_k
  OUTPUT: Ranked list of (alternative, ARS) tuples
  
  candidates <- FindCandidates(drug_to_replace, kg)
  scored_alternatives <- []
  
  FOR EACH candidate IN candidates
    ars <- CalculateARS(drug_to_replace, candidate, regimen, kg)
    scored_alternatives.append((candidate, ars))
  END FOR
  
  sorted_alternatives <- SortByScore(scored_alternatives, descending=True)
  RETURN sorted_alternatives[:top_k]
END FUNCTION
\end{verbatim}

\subsection{S9.1 Clinical Evidence Alignment}

\textbf{Anticoagulant Substitution.} A patient-level meta-analysis of 71,000 patients with atrial fibrillation from major trials, conducted in 2022 by the American Heart Association, showed that substitution of warfarin with direct oral anticoagulants (dabigatran, apixaban, rivaroxaban) was more effective in reducing the risk of stroke or systemic embolism to 3.01\%, compared with 3.69\% for warfarin (hazard ratio [HR] 0.81). This reduced mortality (7.76\% vs. 8.42\%) and significantly decreased intracranial hemorrhage (0.63\% vs. 1.40\%; HR 0.45), while showing a trend toward lower major bleeding (5.05\% vs. 5.94\%; HR 0.86).\textsuperscript{1} This aligned with the substitution recommendations (warfarin to dabigatran, apixaban, rivaroxaban), which indicated relative reductions in adverse outcomes when switching from warfarin to these DOACs. Although evidence showed no significant difference between dabigatran, apixaban, and rivaroxaban in the risk of stroke or systemic embolism, apixaban has been associated with a lower risk of major bleeding compared with both dabigatran and rivaroxaban in multinational cohorts, whereas rivaroxaban often carries a higher bleeding risk relative to dabigatran.\textsuperscript{2,3} This slightly deviates from our findings: in our analysis, switching from warfarin to dabigatran showed a severity reduction of 0.938, a $\Delta$PRI of 0.528, and an ARS of 0.815; switching to apixaban had a severity reduction of 0.383, a $\Delta$PRI of 0.215, and an ARS of 0.333; and switching to rivaroxaban had a severity reduction of 0.313, a $\Delta$PRI of 0.176, and an ARS of 0.272. The discrepancies might have arisen because ARS quantifies reductions in documented drug-drug interactions across the network, rather than intrinsic bleeding risk, which is why dabigatran's lower interaction burden yielded a higher ARS despite apixaban's better bleeding profile.

\textbf{Beta-Blocker Substitution.} Among beta-blockers, atenolol provided the largest severity reduction (0.469), a $\Delta$PRI of 0.200, and an ARS of 0.389, which may be due to its predominant renal excretion and minimal CYP metabolism. Atenolol is primarily eliminated through the kidneys, whereas bisoprolol is cleared approximately half by renal excretion and half via hepatic metabolism. In contrast, metoprolol and carvedilol are largely metabolized by the liver enzyme CYP2D6, which may increase their potential for drug-drug interactions.\textsuperscript{4}

\textbf{Statin Substitution.} The cholesterol guideline from 2018 American College of Cardiology / American Heart Association listed Simvastatin, Lovastatin, and Pravastatin as low to moderate-intensity statins (depending on dose), whereas Atorvastatin is considered moderate to high-intensity based upon lipid lowering potency.\textsuperscript{5} Switching from simvastatin to lovastatin showed a severity reduction of 0.498, $\Delta$PRI of 0.096, and ARS of 0.377; switching to pravastatin yielded identical values; and switching to atorvastatin showed a lower severity reduction (0.233), $\Delta$PRI (0.045), and ARS (0.177). The discrepancy between clinical outcome superiority of atorvastatin\textsuperscript{6,7} and its lower ARS in the network likely reflected a reduction in drug-drug interaction burden rather than an effect on LDL-lowering efficacy or cardiovascular event reduction. Thus, while atorvastatin may provide stronger cardiovascular protection, pravastatin or lovastatin may produce greater interaction-related risk reduction within a drug-drug interaction framework.

\textbf{Clinical Utility Note.} It is important to note that the tool, designed for DDI risk assessment and alternative recommendation, should be used only as a reference and not as a definitive guide for clinical decision-making.

\subsection{S10.1 Input Processing Pipeline}

The web application accepts drug input through three modalities:

\begin{enumerate}
\item \textbf{Direct Entry:} Users enter drug names separated by commas, plus signs (+), semicolons, or newlines. The parser normalizes all separators and removes duplicates while preserving order.

\item \textbf{Narrative Extraction:} Natural language descriptions such as ``I take cardace 5mg in the morning and aspirin at night'' are processed by removing dosage patterns (mg, ml, mcg), frequency terms (once daily, twice a day), and timing phrases (morning, with meals). Remaining words are matched against the drug database using both exact matching and fuzzy string similarity (threshold $\geq$ 0.85 for bigrams, $\geq$ 0.88 for single words).

\item \textbf{OCR Processing:} Prescription images are converted to grayscale and processed with Tesseract OCR. The extracted text is then passed through the narrative extraction pipeline.
\end{enumerate}

\subsection{S10.2 Report Generation}

The comprehensive DDI report includes the following components:

\begin{enumerate}
\item \textbf{Interaction Summary:} All detected DDIs with severity classification (contraindicated, major, moderate, minor) and mechanistic descriptions from the knowledge graph
\item \textbf{Polypharmacy Risk Score:} Combined risk assessment using the PRI formula with weighted contributions from drug count ($C_d$), severity weights ($C_w$), burden score ($C_b$), and semantic similarity ($S$)
\item \textbf{Clinical Summary:} Concise overview of key findings including highest-risk interactions and recommended monitoring
\item \textbf{Per-Drug Risk Index:} Individual PRI scores enabling identification of the highest-contributing drugs to overall regimen risk
\item \textbf{Monitoring Recommendations:} Specific clinical monitoring parameters based on interaction mechanisms (INR for anticoagulant interactions, electrolytes for diuretic combinations)
\item \textbf{Drug Details:} Knowledge graph properties including ATC classification, mechanism of action, and pharmacokinetic parameters
\item \textbf{Alternative Suggestions:} Safer therapeutic alternatives ranked by ARS score from the same pharmacological class
\end{enumerate}
